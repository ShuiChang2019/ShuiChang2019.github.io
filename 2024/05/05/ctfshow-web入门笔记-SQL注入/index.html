\x00-\x7f

\x00-\x7f 表示的是 ASCII 字符集中的字符范围。这个范围包括从十六进制的 00 到 7F 的所有字符，相当于十进制中的 0 到 127。ASCII 字符集包含了控制字符（例如换行符、回车符）、可打印的标点符号、数字和字母等。

具体来说，这个范围包括：

\x00-\x1F 和 \x7F：控制字符，用于文本和打印控制。例如，\x0A（换行符）、\x0D（回车符）等。
\x20-\x7E：可打印字符，包括英文字母（大写和小写）、数字、标点符号以及一些其他符号。

这个范围涵盖了基本的英文字符和常用的控制字符，是计算机编程和数据交换中非常基础的字符编码标准。

sql的弱类型比较

更具体地说，弱类型比较可能出现在使用=操作符比较字符串和数字时。例如，在一些数据库中，字符串’10’和数字10可能被认为是相等的，因为在进行比较前，字符串被自动转换为数字。

sql里，数字和字符串的匹配是弱类型比较，字符串会转换为数字，如0==admin，那么如果输入的username是0，则会匹配所有开头不是数字或者为0的字符串和数字0。（具体的在web187中有提到）

在复现时，我在一个测试表中运行了select username,pass from test where username=0，结果返回了所有的username和pass。因为所有的username字符串开头不是数字或者为0，或是数字0。

password也是弱类型比较。在php中，这段代码会返回ruo。

所以这道题只要输入两个0即可，同时使用了php和sql的弱类型比较。

web195

根据提示，这题是“堆叠注入”。

堆叠注入触发的条件很苛刻,因为堆叠注入原理就是通过结束符同时执行多条sql语句,这就需要服务器在访问数据端时使用的是可同时执行多条sql语句的方法,比如php中mysqli_multi_query()函数,这个函数在支持同时执行多条sql语句,而与之对应的mysqli_query()函数一次只能执行一条sql语句,所以要想目标存在堆叠注入,在目标主机没有对堆叠注入进行黑名单过滤的情况下必须存在类似于mysqli_multi_query()这样的函数,
简单总结下来就是

  1. 目标存在sql注入漏洞
  2. 目标未对”;”号进行过滤
  3. 目标中间层查询数据库信息时可同时执行多条sql语句

web217：屏蔽sleep的时间盲注-BENCHMARK

时间盲注提到，sleep()，BENCHMARK()等亦可用于注入。

BENCHMARK(count,expr) 会重复计算expr表达式count次，通过这种方式就可以评估出mysql执行这个expr表达式的效率。BENCHMARK(9999999,SHA1(‘aaa’))表示运行9999999次’aaa’的SHA1值。

web218：屏蔽sleep的时间盲注-RLIKE

根据wp，可以采用RLIKE运行计算量较大，时间较长的语句来盲注。

rpad(‘a’, 2, ‘a’): 这个函数的作用是对字符串进行右填充，直到达到指定的长度。这里，它将字符串’a’右填充（如果需要）至长度为2，填充字符也是’a’。因为原字符串已经是’a’，且长度为1，所以它将再添加一个’a’到字符串的右侧，最终结果是’aa’。

concat(repeat(‘(a.)+’,2222222),’b’): 这个表达式首先使用repeat(‘(a.)+’, 2222222)重复字符串(a.)+ 2222222次，接着将结果与字符串’b’连接。(a.)+是一个正则表达式，匹配以a开头后跟任意字符的字符串，且+表示前面的模式会被匹配一次或多次。重复这个模式2222222次会生成一个非常长的字符串，最后在这个非常长的正则表达式后面添加一个’b’字符。

select rpad(‘a’,2,’a’) RLIKE concat(repeat(‘(a.*)+’,2222222),’b’): 这个查询检查由rpad(‘a’, 2, ‘a’)生成的字符串’aa’是否匹配由concat(…)生成的正则表达式。基本上，这个正则表达式会非常大，而且在实际执行中可能会导致查询运行非常缓慢或者不可行，因为正则表达式的处理通常是计算密集型的，尤其是当模式非常复杂或匹配的字符串非常长时。
